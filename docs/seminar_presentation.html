LHAMS

Linux Hybrid Mail Audit System

01 · Overview

왜 만들었나

계기. 임베디드 환경을 다루며 시스템·프로그램 오류로 인한 중단(Hang) 또는 무한 루프가 반복적으로 발생했고, 그 원인 분석과 해결의 필요성을 절감했습니다.

목적. 리눅스 서버 내 지정 디렉토리의 파일 변경을 실시간으로 감지하고, 삭제·실행 이력에 대한 감사(Audit) 로그를 구축해 보안과 시스템 안정성을 함께 강화합니다.

파일 변경 실시간 리스트업 삭제 이벤트 별도 기록 실행(execve) 이벤트 추적
Impact

시스템 중단·무한 루프의 원인을 "누가·언제·무엇을" 단위로 즉시 추적 — 장애 대응 시간을 단축합니다.

02 · Requirements

요구사항 분석

기능적 요구사항 Functional

  • 모니터링 대상 경로 하위 전 이벤트를 재귀적(Recursive)으로 감지
  • 로그는 날짜별 로테이션 + 파싱하기 쉬운 포맷(JSON) 유지
  • 이벤트 시각(Timestamp) · 종류(Event Type) · 절대 경로(File Path) 필수 포함

비기능적 요구사항 Non-functional

  • 서버 CPU·메모리 점유율 최소화
  • 파일 수만 개 이상 유입에도 커널 inotify watch 제한 대응
  • 에이전트 비정상 종료 시에도 무중단 재기동 보장

03-1 · Infrastructure

인프라 & OS 튜닝

  • RHEL 8.10 (Ootpa) — 보수적이고 안정적인 엔터프라이즈 환경
  • 대량 메일 파일 유입 시 커널 watch 고갈로 인한 이벤트 누수(No space left) 방지
  • Nginx ↔ 내부 로그 디렉토리 간 SELinux 라벨링 및 포트(80) 개방
/etc/sysctl.conf
$ echo "fs.inotify.max_user_watches=524288" | sudo tee -a /etc/sysctl.conf $ sudo sysctl -p fs.inotify.max_user_watches = 524288
03-2 · Sensors

데이터 수집 · 보안 센서 3종

inotifywait 경량

메일 디렉토리를 재귀 감시하며 1초 이내 이벤트를 캡처. CPU 점유율 최소화.

ClamAV / clamd 스캐너

시그니처를 메모리에 상주(데몬화)시켜 소켓 통신으로 즉시 검사·격리(quarantine).

auditd 커널

execve 시스템 콜을 커널 단에서 가로채 "누가 실행했는가"를 기록. 우회 불가능.

Impact

가벼운 트리거가 대기하다가 이벤트 발생 시에만 무거운 검사를 깨우는 구조 — 관리 리소스를 최소화하면서도 우회 불가능한 커널 기록으로 감사 신뢰성을 확보합니다.

03-3 · Middleware

Python Watchdog — 컨트롤 타워

  • 각 센서가 남긴 텍스트 로그를 실시간 파싱해 정형 JSON(lhams_audit.json)으로 적재
  • TimedRotatingFileHandler로 자정마다 로그 자동 로테이션
  • 화이트리스트/블랙리스트 정규식 기반 필터링, 이상 징후 시 Slack Webhook 호출

systemd 자가 치유

Watchdog 프로세스가 비정상 종료돼도 Restart=always로 5초 이내 강제 재시작 — 24/7 보안 공백을 방지합니다.

Impact

에이전트가 죽어도 시스템이 스스로 복구되므로, 야간·주말에도 감시 공백이 발생하지 않습니다.

03-4 · Frontend & Web

시각화 · 웹 서비스

Node.js 20.x

RHEL 8 기본 10.x 잠금을 해제하고 최신 런타임으로 완전 마이그레이션.

06 · Practice 3

realtime_monitor.sh — 핵심 파이프라인

realtime_monitor.sh
inotifywait -m -r -e create,close_write,moved_to \ --exclude '\.(swp|swx|swpx)$|/\.' "$WATCH_DIR" --format '%w%f' | while read FILE do SCAN_RESULT=$(sudo clamdscan --stream --fdpass --no-summary "$FILE") # FOUND 감지 시 자동 격리(quarantine) done

--fdpass

파일 디스크립터를 직접 전달해 권한 문제 없이 검사

--no-summary

요약 정보 없이 검사 결과만 간결하게 출력

08 · Backend Agent

Python Watchdog — JSON 로깅 구조

lhams_audit.json 필드
{ "timestamp": "2026-07-10 21:42:45", "event_type": "MALWARE_DETECTED", "file_path": "/mail/.../malware_test.com", "user": "www-data", "risk_level": "High" }
  • on_created / on_modified / on_deleted 훅으로 전 이벤트 포착
  • 생성·수정 시 scan_malware()가 clamdscan 호출 → 탐지 시 risk_level 자동 격상
  • 최신 100건만 유지하며 파일 소유자(pwd)까지 함께 기록
